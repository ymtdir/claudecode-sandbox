# 要件定義書

## プロダクト概要

### 名称

**ミニマル家計簿** - iOS 標準アプリ風の必要最低限な家計簿

### プロダクトコンセプト

- **必要最低限の機能のみ**: カレンダー入力・カテゴリ管理・月次サマリ以外の機能を意図的に持たない（ミニマリスト思想）
- **iOS 標準アプリ準拠の UI/UX**: カレンダー / リマインダーアプリと同じ操作感・見た目で、学習コストゼロで使える
- **シンプルなローカルアプリ**: 端末ローカルで完結。クラウド同期・アカウント・広告なし

### プロダクトビジョン

既存の家計簿アプリは多機能だが、UI が独自すぎたり広告が多かったりして、iOS 標準アプリのような心地よさを感じにくい。本アプリは、Apple 純正アプリと並べて違和感のないデザイン・操作感で、機能を必要最低限まで絞り込み、自分の支出をカレンダー上で素直に振り返れるツールを目指す。第一義は開発者本人の日常利用だが、同じ嗜好を持つユーザーに使ってもらえる品質を達成できれば、将来的な App Store 公開も視野に入れる。

### 目的

- 自分が普段使いできる、iOS 標準 UI/UX に揃えたミニマルな家計簿アプリを完成させる
- Swift / SwiftUI を使った iOS アプリ開発を、実機で動作させるレベルまで実践的に習得する
- 収支データの永続化（端末ローカル）の実装パターンを習得する
- 将来的な App Store 公開を選択肢として残せる品質・構成を維持する

## ターゲットユーザー

### プライマリーペルソナ: 開発者本人（個人利用）

- iOS / macOS を日常的に使うユーザー
- 既存の家計簿アプリの UI に違和感や煩雑さを感じ、自分好みのシンプルなものが欲しい
- 現在の課題: 純正アプリのような操作感を持つ家計簿アプリが見当たらない / 余計な機能や広告が邪魔
- 期待する解決策: 標準アプリと並べて違和感のない、必要最小限の家計簿
- 典型的なワークフロー: 支出があったタイミング、または 1 日の終わりにアプリを開き、カレンダー上の該当日に金額・カテゴリ・メモを入力する。月末にカレンダー上部の月次サマリで合計を確認する

### セカンダリーペルソナ: ミニマル志向の iOS ユーザー

- 既存の家計簿アプリの多機能性・広告・独自 UI に違和感を持っている
- 自分にとって本当に必要な記録・集計だけを iOS 標準アプリ風の操作感で行いたい
- プライマリとの違い: 開発に関与せず、純粋に利用者として App Store 等を経由してアプリを入手することを想定
- 想定獲得経路: 将来的な App Store 公開時の自然流入（積極的なマーケティングは想定しない）

## 成功指標(KPI)

商用プロダクトではなく、開発者本人の利用と将来的な公開可能性を達成基準とする。

- 実機の iOS 端末（開発者本人の iPhone）でアプリがインストール・起動・基本操作できる（プロジェクト完了時点）
- MVP に含めるすべての P0 機能が動作する（プロジェクト完了時点）
- 開発者本人が、リリース後 1 ヶ月間アプリを継続使用できる（実用性の自己検証）
- 将来的な App Store 公開を判断できる品質に到達する: クラッシュなし・データ消失なしで 1 ヶ月の通常利用に耐える

## 機能要件

### コア機能(MVP)

#### カレンダー表示

**ユーザーストーリー**:
ユーザーとして、いつ・いくら使ったかを日付軸で振り返るために、月カレンダーで各日の収支サマリを一覧表示したい。

**受け入れ条件**:

- [ ] 起動時に当月のカレンダーがメイン画面として表示される
- [ ] 各日付セルに、その日の支出合計（または収支差）が小さく表示される
- [ ] ヘッダー操作または横スワイプで前月・翌月に切り替えできる
- [ ] iOS 標準カレンダーアプリと違和感のない見た目（フォント・余白・色）で表示される

**優先度**: P0(必須)

#### 日別の収支詳細表示

**ユーザーストーリー**:
ユーザーとして、特定日の支出内訳を確認するために、カレンダーの日付をタップしてその日の収支一覧を見たい。

**受け入れ条件**:

- [ ] カレンダーの日付セルをタップすると、その日の収支一覧画面に遷移する
- [ ] 一覧には各レコードの金額・カテゴリ・メモが表示される
- [ ] 一覧から個別レコードを選択して編集・削除できる
- [ ] 一覧画面から新規レコードの追加導線がある

**優先度**: P0(必須)

#### 収支の入力（追加・編集・削除）

**ユーザーストーリー**:
ユーザーとして、その日の支出または収入を記録するために、金額・カテゴリ・日付・メモを入力したい。

**受け入れ条件**:

- [ ] 金額・種別（収入 / 支出）・カテゴリ・日付・メモ（任意）を入力できる
- [ ] 既存レコードを編集・削除できる
- [ ] 入力内容は端末ローカルに永続化され、アプリ再起動後も保持される
- [ ] 必須項目（金額・カテゴリ・日付）が未入力の場合は保存できない

**優先度**: P0(必須)

#### カテゴリ管理

**ユーザーストーリー**:
ユーザーとして、自分の生活に合ったカテゴリで支出を分類するために、カテゴリの追加・編集・削除をしたい。

**受け入れ条件**:

- [ ] 初期状態でいくつかの一般的なカテゴリ（食費・交通費 等）が登録されている
- [ ] カテゴリの追加・名称変更・削除ができる
- [ ] カテゴリは収支入力画面の選択肢として利用される
- [ ] 既存レコードで使用中のカテゴリを削除する場合の挙動が定義されている（削除不可、または既存レコードのカテゴリを未分類化、いずれかを実装）

**優先度**: P0(必須)

#### 月次サマリ表示

**ユーザーストーリー**:
ユーザーとして、その月にいくら使ったかを把握するために、月の合計収入・合計支出・カテゴリ別集計を確認したい。

**受け入れ条件**:

- [ ] カレンダー画面の上部または専用領域に、表示中の月の合計収入・合計支出・差額が表示される
- [ ] カテゴリ別の支出合計を確認できる（同画面 or 遷移先のいずれか）
- [ ] 月を切り替えると、サマリも該当月の値に追従する

**優先度**: P0(必須)

## 非機能要件

### パフォーマンス

- アプリ起動から当月カレンダー表示完了までが 2 秒以内（実機）
- カレンダーの月切り替え操作の応答が 0.5 秒以内（実機）

### ユーザビリティ

- iOS 標準のカレンダー / リマインダーアプリと一貫した見た目・操作感（システムフォント・SF Symbols・標準コンポーネントを使用）
- ライト / ダークモード両方に対応する
- Dynamic Type（システムフォントサイズ設定）に追従する

### 信頼性

- ローカル DB に保存したレコードはアプリ強制終了・端末再起動後も失われない
- 入力中のデータの保存失敗時はユーザーに明示する（保存ボタン押下時の即時フィードバック）

### セキュリティ

- ネットワーク通信を行わない（外部送信なし）
- 端末外部にデータを書き出さない（OS 標準のバックアップ経路を除く）

### 対応環境

- iOS バージョン: 開発時点で SwiftUI の主要機能が安定して使える最新の 1〜2 メジャーバージョン
- 想定デバイス: iPhone（iPad は対象外、ただし動作禁止はしない）

## スコープ外

### 完全にスコープ外（個人ユース完結方針のため恒久的に対象外）

- 複数ユーザー / 認証 / アカウント機能
- 共同家計など複数人でのデータ共有
- クラウド同期（端末間でのデータ共有）
- 広告・課金・分析 SDK の組み込み

### 短期スコープ外（MVP では対象外、将来検討の余地あり）

- App Store への公開・配布（品質基準を満たした段階で検討）
- iCloud バックアップ連携（同期ではなく単一端末のバックアップ目的）
- データのエクスポート（CSV 等）
- 予算設定・予算超過アラート
- グラフ表示（円グラフ・棒グラフ等）
- 複数口座 / 複数通貨対応
- 通知 / リマインダー機能
