# 要求内容

## 概要

UnifiedCalアプリで使用する共通UIコンポーネントライブラリを導入し、デザインシステムの基盤を構築します。React Native ElementsやGesture Handlerなど、必要なUIライブラリを設定します。

## 背景

- 統一されたデザインシステムがないため、コンポーネント間で一貫性が保てない
- 共通UIコンポーネントがないため、同じようなコードを重複して書いている
- モバイルアプリに最適化されたインタラクション（スワイプ、ジェスチャー）が実装できていない
- カレンダー機能の実装に必要な専用ライブラリが導入されていない

## 実装対象の機能

### 1. UIライブラリのインストールと設定
- React Native Elements、Vector Icons、Calendarsなどの必要なライブラリをインストール
- Gesture HandlerとReanimatedの設定
- Safe Area Contextの導入

### 2. デザインテーマの設定
- カラーテーマ（プライマリー、セカンダリー、成功、警告色など）の定義
- タイポグラフィ（見出し、本文、キャプション）の設定
- スペーシングとレイアウトの標準化

### 3. 共通UIコンポーネントの実装
- Button コンポーネント（プライマリー、セカンダリー、無効状態）
- Card コンポーネント（影、境界線、パディング）
- Input コンポーネント（テキスト入力、エラー表示）
- Header コンポーネント（タイトル、戻るボタン、アクション）
- TaskCard コンポーネント（タスク情報表示、スワイプアクション）
- CalendarDay コンポーネント（日付表示、選択状態）

### 4. アイコンシステムの構築
- Material Iconsの設定
- よく使うアイコンのラッパーコンポーネント作成

### 5. アニメーション基盤の設定
- React Native Reanimatedの初期設定
- 基本的なアニメーションユーティリティの作成

## 受け入れ条件

### UIライブラリ
- [ ] 必要なライブラリがすべてインストールされている
- [ ] iOS/Android両方でライブラリが正しく動作する
- [ ] TypeScript型定義が適用されている

### デザインシステム
- [ ] カラーテーマが定義され、一元管理されている
- [ ] タイポグラフィが定義され、再利用可能になっている
- [ ] テーマの変更が容易にできる構造になっている

### 共通コンポーネント
- [ ] 基本的な共通コンポーネントが実装されている
- [ ] 各コンポーネントが開発ガイドラインに準拠している
- [ ] コンポーネントのPropsがTypeScriptで型定義されている
- [ ] スタイルが一貫性を持っている

### アイコン・アニメーション
- [ ] アイコンが正しく表示される
- [ ] アニメーションライブラリが動作する
- [ ] パフォーマンスに問題がない

## 成功指標

- すべての共通コンポーネントが再利用可能な形で実装される
- コンポーネントを組み合わせて画面を構築できる
- デザインの一貫性が保たれる
- TypeScriptによる型安全性が確保される

## スコープ外

以下はこのフェーズでは実装しません:

- 複雑なアニメーション効果
- テーマの動的切り替え機能
- ダークモード対応
- Storybookの設定（別タスクで実施）
