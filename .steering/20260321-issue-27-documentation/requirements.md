# 要求内容

## 概要

プロジェクトの開発・運用・保守に必要なドキュメントを整備し、開発者がプロジェクトを理解しやすくする。

## 背景

Issue #27ではReact Native（UnifiedCal）向けのドキュメント整備を想定していますが、現在のプロジェクトはReact + Viteのテストプロジェクトです。そのため、現在のプロジェクト構成に適した形でドキュメントを整備します。

既存のdocs/フォルダにはスペック駆動開発の永続ドキュメント（PRD、機能設計、アーキテクチャ等）が存在しますが、これらはReact Nativeを想定したものです。今回は以下を実施します：

1. 既存ドキュメントを現在のプロジェクト構成（React + Vite）に合わせて更新
2. 不足している開発者向けドキュメントを追加

## 実装対象の機能

### 1. トラブルシューティングガイド

- よくある問題と解決方法
- ビルドエラーの対処法
- 開発サーバーの問題
- テスト実行時の問題

### 2. コントリビューションガイド

- コーディング規約の詳細
- Gitワークフロー
- PR作成のガイドライン
- コードレビューのポイント

### 3. APIリファレンス（TypeDoc）

- 自動生成APIドキュメント
- コンポーネントのProps定義
- ユーティリティ関数の仕様
- Hookの使用方法

### 4. セキュリティガイドライン

- 機密情報の管理
- 依存関係のセキュリティ
- 脆弱性への対応手順

### 5. README.mdの強化（必要に応じて）

- プロジェクト概要の明確化
- セットアップ手順の詳細化
- トラブルシューティングリンク追加

## 受け入れ条件

### トラブルシューティングガイド

- [ ] `docs/TROUBLESHOOTING.md`が作成されている
- [ ] よくある問題とその解決方法が記載されている
- [ ] 各問題に対する具体的なコマンド例がある

### コントリビューションガイド

- [ ] `docs/CONTRIBUTING.md`が作成されている
- [ ] コーディング規約が明確に記載されている
- [ ] Gitワークフローが説明されている
- [ ] PR作成手順が記載されている

### APIリファレンス

- [ ] TypeDocの設定ファイル（`typedoc.json`）が作成されている
- [ ] `npm run docs:generate`でAPIドキュメントが生成される
- [ ] 生成されたドキュメントが`.gitignore`に追加されている

### セキュリティガイドライン

- [ ] `docs/SECURITY.md`が作成されている
- [ ] 機密情報管理のベストプラクティスが記載されている
- [ ] 脆弱性報告の手順が明記されている

### README.md強化

- [ ] 必要に応じてREADME.mdが更新されている
- [ ] 新しいドキュメントへのリンクが追加されている

## 成功指標

- 新しい開発者がドキュメントを読んで、環境構築とコントリビューションができる
- よくある問題がドキュメントで解決できる
- APIドキュメントが自動生成され、最新の状態を保てる

## スコープ外

以下はこのフェーズでは実装しません：

- デプロイガイド（現在はテストプロジェクトのため不要）
- リリースノートテンプレート（まだリリースがないため）
- React Native固有のドキュメント（プロジェクト構成と異なるため）
- ユーザー向けドキュメント（開発者向けに特化）
