# 要求内容

## 概要

GitHub Actionsを使用したCI/CD環境を構築し、自動テスト、ビルド、デプロイのパイプラインを実装する。

## 背景

現在、プロジェクトには基本的なCIワークフロー（フォーマット、リント、ビルドチェック）のみが存在している。プロジェクトの品質を継続的に保ち、開発効率を向上させるために、より包括的なCI/CD環境が必要である。

Issue #26では、React Native（UnifiedCal）向けのCI/CDを想定しているが、現在のプロジェクトはReact + Viteのプロジェクトであるため、現在のプロジェクト構成に適した形で実装する。

## 実装対象の機能

### 1. テストワークフロー
- 自動テスト実行（Vitest）
- コードカバレッジレポート生成
- テスト結果のアーティファクト保存

### 2. コード品質チェックワークフロー
- 既存のフォーマット・リント・ビルドチェックを強化
- 型チェック（TypeScript）の追加
- 複数Node.jsバージョンでのテスト

### 3. 依存関係の自動更新
- Dependabotの設定
- npm依存関係の週次チェック
- GitHub Actionsの週次チェック

### 4. セキュリティスキャン
- npm auditの実行
- 脆弱性レポートの生成

### 5. リリースワークフロー（オプション）
- タグベースのリリース自動化
- リリースノート自動生成
- ビルド成果物のアップロード

## 受け入れ条件

### テストワークフロー
- [ ] プッシュ時に自動テストが実行される
- [ ] PRマージ前にテストが通ることが必須になっている
- [ ] コードカバレッジレポートが生成される
- [ ] テスト結果がPRに表示される

### コード品質チェック
- [ ] フォーマットチェックが実行される
- [ ] リントチェックが実行される
- [ ] 型チェックが実行される
- [ ] ビルドチェックが実行される
- [ ] 複数Node.jsバージョン（20.x, 25.x）でテストされる

### 依存関係の自動更新
- [ ] Dependabotが設定されている
- [ ] 週次で依存関係がチェックされている
- [ ] Pull Requestが自動作成される

### セキュリティスキャン
- [ ] npm auditが定期実行される
- [ ] 脆弱性が発見された場合に通知される

## 成功指標

- mainブランチへのマージ前に品質チェックが自動実行される
- テストカバレッジが可視化される
- 依存関係の更新が自動化される
- セキュリティ問題が早期に検出される

## スコープ外

以下はこのフェーズでは実装しません:

- デプロイ自動化（Netlify/Vercelは独自のCI/CDを持つため）
- パフォーマンステスト
- E2Eテスト（Playwrightなど、現在のプロジェクトにはまだ存在しない）
- Docker環境での実行
- React Native固有のビルド（iOS/Android）
