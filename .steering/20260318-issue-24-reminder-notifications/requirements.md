# 要求仕様書

## Issue情報
- Issue番号: #24
- タイトル: リマインダー通知機能 - プッシュ通知の実装
- ラベル: enhancement, feature

## 背景と目的
タスクのリマインダー機能を実装し、指定した時刻にプッシュ通知を送信する機能を構築する。
ユーザーがタスクの期限を忘れないように、事前通知やスヌーズ機能を提供する。

## 機能要件

### 基本機能
1. 通知権限のリクエストと管理
2. 時間ベースのリマインダー設定（時刻通知、事前通知）
3. 繰り返し通知の設定（日次、週次、月次）
4. 通知からのタスク完了操作
5. スヌーズ機能（5分後、10分後の再通知）

### 通知タイプ
- **時刻通知**: タスクの予定時刻に通知
- **事前通知**: 10分前、30分前、1時間前に通知
- **繰り返し通知**: 定期的なタスクのリマインダー

## 非機能要件

### パフォーマンス
- 通知は指定時刻から30秒以内に配信
- バックグラウンドでも確実に通知が動作

### 互換性
- iOS/Android両対応（React Native Web環境）
- ブラウザの通知APIを使用

### ユーザビリティ
- 直感的な通知設定UI
- ワンタップでタスク完了/スヌーズ

## 受け入れ条件
- [ ] 通知権限のリクエストが表示される
- [ ] 時間ベースのリマインダーが設定できる
- [ ] 指定時刻に通知が届く
- [ ] 事前通知（10分前、30分前など）が設定できる
- [ ] 繰り返し通知が設定できる
- [ ] 通知から直接タスクを完了できる
- [ ] スヌーズ機能が動作する
- [ ] バックグラウンドでも通知が届く

## 制約事項
- React Native Webのため、ネイティブの通知ライブラリは使用不可
- ブラウザのNotification APIを使用する必要がある
- PWA化によるバックグラウンド通知の実装

## スコープ外
- 位置情報ベースの通知（ジオフェンシング）- Web環境では制限あり
- FCM（Firebase Cloud Messaging）- 初期実装では不要
- リモートプッシュ通知 - ローカル通知のみ実装
